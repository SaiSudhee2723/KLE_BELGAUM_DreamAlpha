## Problem Statement

India has **1 billion+ rural citizens** and only **1 doctor per 10,000 people** in remote areas. ASHA (Accredited Social Health Activist) workers are the frontline healthcare providers — but they work with paper forms, zero diagnostic tools, and no real-time guidance.

**The result:** Diseases like malaria, dengue, TB, and anaemia go undetected for weeks. Pregnant women miss critical ANC checkups. Outbreaks spread before anyone notices.

---

## Solution Overview

Sahayak AI is a **voice-first, offline-capable medical assistant** built for ASHA workers. It runs on the **AMD Ryzen AI NPU** (Neural Processing Unit), enabling real-time AI inference at 4.2W — without any internet connection.

Workers speak in **Kannada, Hindi, or English**. The system transcribes, analyzes against 12 ICMR clinical protocols, and delivers structured diagnoses with action items — all in under 200ms.

---

## Key Features

### For ASHA Workers
- **Voice Diagnosis** — speak symptoms in any supported language, get structured AI diagnosis
- **4-Tier LLM Chain** — Groq (primary) → Gemini (fallback) → HuggingFace → Rule-based engine
- **12 ICMR Disease Protocols** — malaria, dengue, TB, diabetes, anaemia, hypertension, cholera, typhoid, pneumonia, diarrhea, COVID-19, maternal complications
- **FAISS RAG Engine** — retrieval-augmented generation over Indian clinical guidelines
- **Outbreak Heatmap** — real-time disease visualization by district
- **Gamification** — badges, streaks, and impact scores to motivate workers
- **Proactive AI Briefing** — daily priority list generated by `/agent/proactive`
- **Government Reports** — auto-fill HMIS monthly reports from analytics

### For Patients
- **Health Dashboard** — animated health score, vitals sparklines, risk badge
- **PDF/Image Upload** — AI extracts vitals automatically from medical reports
- **Share Code System** — patients control which doctors can see their records
- **Voice Appointment Booking** — conversational AI books doctor slots via phone (VAPI integration)

### For Doctors
- **Patient Grid** — all accessible patients with risk levels and last report dates
- **Share Code Access** — enter patient-provided code to unlock full record
- **Vitals Trend Charts** — Recharts timeline across all visits
- **Appointment Management** — calendar view with available slots

### Platform
- **PWA + Offline** — IndexedDB queue syncs when reconnected
- **Firebase Auth** — Google OAuth + email/password for all 3 roles
- **Make.com Automations** — automated SMS alerts, email notifications, VAPI calls
- **AMD NPU** — model quantized to 4-bit INT, runs on Ryzen AI at 50 TOPS / 4.2W

## Local Setup

### Prerequisites

- Python 3.11+
- Node.js 18+
- Git

### 1. Clone the Repository

```bash
git clone https://github.com/Praful0306/DreamAlpha_For_Amd.git
cd DreamAlpha_For_Amd
```

### 2. Backend Setup

```bash
cd sahayak

# Create virtual environment
python -m venv .venv
source .venv/bin/activate      # Linux/Mac
# OR: .venv\Scripts\activate   # Windows

# Install dependencies
pip install -r requirements.txt

# Create .env file (see Environment Variables section below)
cp .env.example .env
# Edit .env with your actual API keys

# Run database migrations
python migrate_db.py

# Start the backend server
uvicorn main:app --reload --port 8000
```

Backend will be available at: `http://localhost:8000`  
API docs (Swagger UI): `http://localhost:8000/docs`

### 3. Frontend Setup

```bash
cd sahayak-frontend

# Install dependencies
npm install

# Create .env.local (see Environment Variables section below)
cp .env.local.example .env.local
# Edit with your Firebase config

# Start the dev server
npm run dev
```

Frontend will be available at: `http://localhost:5173`

> **Note:** The Vite dev server proxies all `/api/*` requests to `http://localhost:8000`, so no CORS issues in development.

### 4. Verify Everything Works

```bash
# Check backend health
curl http://localhost:8000/health
# Expected: {"status":"healthy","version":"3.3.1"}

# Check frontend proxy
curl http://localhost:5173/api/health
# Expected: same response (proxied)
```

## Deployment

### Backend — Render (Free Tier)

1. Push your code to GitHub.
2. Go to [render.com](https://render.com) → New → Web Service.
3. Connect your GitHub repo.
4. Configure:
   - **Build Command:** `pip install -r requirements.txt && python migrate_db.py`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`
   - **Root Directory:** `sahayak`
5. Add environment variables in the Render dashboard (all keys from `.env`).
6. Set `FRONTEND_URL` to your Vercel URL once deployed.
7. Deploy — your API will be at `https://sahayak-api.onrender.com`.

Alternatively, the included `sahayak/render.yaml` automates this config:

```yaml
services:
  - type: web
    name: sahayak-api
    env: python
    rootDir: sahayak
    buildCommand: pip install -r requirements.txt && python migrate_db.py
    startCommand: uvicorn main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: FRONTEND_URL
        value: https://sahayak-ai.vercel.app
      - key: GROQ_API_KEY_1
        sync: false
      - key: GEMINI_API_KEY_1
        sync: false
      - key: SECRET_KEY
        sync: false
      - key: FIREBASE_CREDENTIALS_JSON
        sync: false
```

### Frontend — Vercel (Free Tier)

```bash
cd sahayak-frontend

# Install Vercel CLI
npm i -g vercel

# Set production API URL
echo "VITE_API_URL=https://sahayak-api.onrender.com" > .env.production

# Deploy
vercel --prod
```

Or connect your GitHub repo in the Vercel dashboard for automatic deploys on every push.

**Required Vercel environment variables:**
- `VITE_API_URL` = `https://sahayak-api.onrender.com`
- All `VITE_FIREBASE_*` variables
- All `VITE_MAKECOM_*` webhooks

## Database Schema

### Core Tables (SQLite)

```sql
-- Users (all roles)
users
  id            INTEGER PRIMARY KEY
  firebase_uid  TEXT UNIQUE
  email         TEXT UNIQUE
  full_name     TEXT
  role          TEXT  -- 'patient' | 'doctor' | 'asha'
  password_hash TEXT
  specialization TEXT  -- doctors only
  hospital      TEXT   -- doctors only
  district      TEXT   -- asha only
  created_at    DATETIME

-- Patients (linked to patient-role users)
patients
  id              INTEGER PRIMARY KEY
  user_id         INTEGER FK -> users.id
  name            TEXT
  age             INTEGER
  gender          TEXT
  phone           TEXT
  village         TEXT
  district        TEXT
  blood_group     TEXT
  medical_history TEXT
  is_pregnant     BOOLEAN
  share_code      TEXT UNIQUE   -- 8-char code for doctor access
  share_code_active BOOLEAN
  asha_worker_id  INTEGER FK -> users.id
  created_at      DATETIME

-- Medical Reports
medical_reports
  id              INTEGER PRIMARY KEY
  patient_id      INTEGER FK -> patients.id
  report_title    TEXT
  report_type     TEXT
  bp              TEXT    -- "120/80"
  hr              INTEGER
  temp            FLOAT
  spo2            FLOAT
  sugar_fasting   FLOAT
  sugar_post      FLOAT
  hemoglobin      FLOAT
  creatinine      FLOAT
  cholesterol     FLOAT
  weight_kg       FLOAT
  symptoms        TEXT
  diagnosis       TEXT
  medications     TEXT
  risk_level      TEXT
  ai_risk_level   TEXT
  ai_analysis     TEXT
  ai_summary      TEXT
  is_ai_extracted INTEGER
  created_at      DATETIME

-- Doctor-Patient Access
doctor_patient_access
  id          INTEGER PRIMARY KEY
  doctor_id   INTEGER FK -> users.id
  patient_id  INTEGER FK -> patients.id
  is_active   BOOLEAN
  granted_at  DATETIME

-- Checkups (doctor visit records)
checkups
  id            INTEGER PRIMARY KEY
  patient_id    INTEGER FK -> patients.id
  checkup_date  DATE
  next_checkup  DATE
  doctor_name   TEXT
  hospital      TEXT
  reason        TEXT
  findings      TEXT
  medications   TEXT

-- Appointments
appointments
  id            INTEGER PRIMARY KEY
  doctor_id     INTEGER FK -> users.id
  patient_id    INTEGER FK -> patients.id (nullable)
  patient_name  TEXT
  phone         TEXT
  date          TEXT
  time_slot     TEXT
  reason        TEXT
  status        TEXT  -- 'scheduled' | 'completed' | 'cancelled'
  is_manual     BOOLEAN
  created_at    DATETIME
```

## AMD NPU Integration

Sahayak AI is specifically optimized for the **AMD Ryzen AI NPU** (Neural Processing Unit) found in Ryzen 7000/8000 series processors.

### Why AMD NPU?

| Feature | Cloud GPU | AMD Ryzen AI NPU |
|---------|-----------|-----------------|
| Power Consumption | 80W+ | **4.2W** |
| Internet Required | Yes | **No** |
| Latency (P50) | 800ms+ | **180ms** |
| Data Privacy | Cloud-stored | **100% local** |
| Cost | $0.002/query | **Zero** |
| Works in Rural Areas | No | **Yes** |

### Model Optimization

- **Quantization:** 4-bit INT (GGUF format) via llama.cpp
- **NPU Acceleration:** ONNX Runtime with AMD VAIML EP (Vitis AI ML Execution Provider)
- **Throughput:** 50 TOPS on Ryzen AI 300 series
- **Model Size:** 4GB RAM footprint for llama3-8b-4bit

### Setup NPU Inference (Local)

```bash
# Install AMD ROCm (Linux) or ONNX Runtime EP (Windows)
pip install onnxruntime-directml   # Windows NPU acceleration

# Or use llama.cpp with AMD GPU offload
pip install llama-cpp-python --extra-index-url https://abetlen.github.io/llama-cpp-python/whl/cpu
```

The `services/llm_chain.py` detects NPU availability automatically and routes to the local model when available, falling back to Groq API otherwise.
